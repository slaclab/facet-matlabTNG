The LUCRETIA Project

TESLA Benchmarking Test

For each of these examples the original lattice file (bmad format) may be found here, and the wakefield data here courtesy of Jeff Smith.

Results from benchmarking other codes are also on Jeff's webspace.

Since bmad format is not exactly xsif compliant, these files had to be edited slightly for acceptance by XSIFToLucretia. The edited files can be found in the directories linked to in each of the examples below.

Example 1 :  This example starts with a perfectly aligned linac, offsets the incoming beam by 5 microns in y, and tracks it through to the end. A switch at the beginning of the script determines whether a sparse or macro-particle representation of the beam is used. The tracking is performed with all four combinations of the longitudinal and transverse wakefields being on and off.
The x and y beta functions, as well as the y dispersion, is plotted in one figure window, while the y beam position and normalised emittance is plotted (for all four wakefield conditions) in a second window.

Example 2 :  This script starts by misaligning the lattice according to this file. PT applied DFS to this beamline, and determined that the corrector settings from this file should be applied.
Once these misalignments and corrector settings are applied, the script tracks the beam through the linac under varying wakefield conditions.
